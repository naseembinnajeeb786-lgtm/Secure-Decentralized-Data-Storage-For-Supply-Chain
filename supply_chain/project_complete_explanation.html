What is blockchain.py?
It is a Python program that creates a digital notebook for a product's journey through a supply chain. Every step of the journey — from raw materials to final delivery — is stored as a "block". Each block is locked with a mathematical fingerprint (SHA-256 hash). If anyone secretly changes an old record, the fingerprint breaks and Python catches it immediately.
Block class
One supply chain record. Stores the event, timestamp, and SHA-256 hash. Created automatically when you call add_block().
Blockchain class
Manages all blocks as a Python list. Links each block to the previous one using its hash. Validates the whole chain.
Main program
Creates 5 blocks, prints them, checks validity, then tampers with Block 2 to prove tamper detection.

Step-by-step working of your code
1
Program starts — Python loads hashlib and datetime modules. No installation needed — both are built into Python.
2
Blockchain() is created — The constructor runs automatically and creates Block 0 (Genesis Block) with previous_hash = "0" and calculates its SHA-256 hash.
3
add_block() is called 4 times — Each call creates a new Block. The new block takes the previous block's hash as its own "previous_hash", then calculates its own SHA-256 hash. This is what creates the chain.
4
print_chain() prints all blocks — Shows block number, data (event, location, qty), timestamp, previous hash (40 chars), and current hash (40 chars).
5
is_valid() returns True — Loops through every block and checks: (1) stored hash matches freshly computed hash, (2) previous_hash matches the actual previous block's hash. All pass → True.
6
Block 2 is tampered — The code directly changes Block 2's qty from 480 to 200 in memory. But Block 2's self.hash is NOT updated — it still holds the old hash.
7
is_valid() returns False — When it checks Block 2, it recalculates the hash from the new data (qty=200). The result is DIFFERENT from the stored hash (which was computed with qty=480). Mismatch detected → False.
8
Block 3 and 4 show BROKEN LINK — Block 3 stored the OLD hash of Block 2 as its previous_hash. Now Block 2's hash has changed, so Block 3's link no longer matches. Same for Block 4.
How SHA-256 hashing works in your code
text = str(index) + timestamp + str(data) + previous_hash hash = hashlib.sha256(text.encode()).hexdigest() # Result: "8dce55a20f3118b74a1f92e7c6d83051..." (64 characters)
All four values are joined into one string → converted to bytes with .encode() → SHA-256 produces a unique 64-character fingerprint. Change any part of the input → the output changes completely.
Real-world meaning of your project
Your project proves a fundamental concept used in Bitcoin, Ethereum, and real corporate supply chains like IBM Food Trust, Walmart's food tracking, and Maersk's shipping blockchain. The concept is: you cannot secretly change past records in a blockchain.
Without blockchain (old way)
A corrupt warehouse manager changes "qty: 480" to "qty: 200" in the database. Steals 280 units. Nobody notices. No proof it was changed. No audit trail.
With blockchain (your project)
The same change is made to Block 2. is_valid() immediately detects "HASH MISMATCH" at Block 2. The fraud is caught. The exact block is identified. The chain proves it was tampered.
